# Live Band Timer (PWA) ⏱️🎸

Googleスプレッドシートのタイムテーブルとリアルタイムに連動する、イベント・ステージ用のカウントダウンタイマーです。
PWA（Progressive Web App）に対応しており、一度読み込めば地下などの**圏外（オフライン）環境でも動作**します。

## 🚀 主な機能
- **簡単な本番オペレーション**: 各バンドの終了時間を事前に設定しておけば、当日のタイムキーパーは**バンドの演奏が終わる度に「次のバンドへ」ボタンを押すだけ**で進行できます。
- **安心のループ・リカバリー**: 万が一「次のバンドへ」を押しすぎてしまっても、**出演バンドの数だけクリック（タップ）を続ければ、自動的に一周して最初のバンドに戻る**仕様のため、現場での誤操作も簡単にリセットできます。
- **視覚的なタイムオーバー警告**: 設定した終了時刻をオーバーすると、タイマーが自動的に**赤文字**に切り替わり、`-01:23` のように超過時間をマイナスカウント（経過カウント）し続けます。これにより、イベントが現在何分何秒押している（遅れている）のかが一目で把握できます。
- **GAS（Google Apps Script）連動**: スプレッドシートを書き換えるだけで、タイマー側のバンド名や終了時刻が自動更新されます。
- **オフライン対応**: サービスワーカー（`sw.js`）と LocalStorage の二段構えで、電波がない場所でも前回のデータで起動します。
- **フルスクリーン＆常時点灯**: ライブ中の視認性を最優先し、画面の自動スリープを防止、タップで全画面表示になります。
- **タイム調整機能**: 全バンドのスケジュールを「一発でプラスマイナス〇分」オフセット（ズレ調整）できます。

## 🛠️ システム構成・ファイル構造
このリポジトリは以下のファイルで構成されています。

- `index.html` : タイマーの本体（HTML/CSS/JavaScript）
- `manifest.json` : アプリとしてスマホにインストールするための設定ファイル
- `sw.js` : オフライン起動を可能にするサービスワーカー
- `icon-192.png` / `icon-512.png` : ホーム画面用のアプリアイコン画像

## 📋 運用・セットアップマニュアル

### 1. Googleスプレッドシートの準備
A列に「終了時刻（HH:MM）」、B列に「バンド名」を入力します。1行目は Time / Band でも構いません。

| 終了時刻 | バンド名 |
| :--- | :--- |
| 13:30 | 深紫伝説 |
| 14:00 | ペトロールス |

### 2. GAS（Google Apps Script）側のコード
スプレッドシートの「拡張機能」＞「Apps Script」を開き、以下のコードを貼り付けます。
※見た目の時間通りに文字として取得するため `.getDisplayValues()` を使用しています。

```javascript
function doGet() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  const data = sheet.getDataRange().getDisplayValues(); // 画面の表示のまま取得
  const jsonOutput = ContentService.createTextOutput(JSON.stringify(data));
  return jsonOutput.setMimeType(ContentService.MimeType.JSON);
}
```

【重要】 右上の「デプロイ」＞「新しいデプロイ」から、種類を「Webアプリ」、アクセスできるユーザーを「全員（Anyone）」にしてデプロイし、発行されたURLを index.html の GAS_URL に設定してください。

index.html 106行目付近です、GASのデプロイで発行された　https://ではじまりexecで終わるURLに置き換えて使ってください
```
    // -----------------------------------------------------------------
    // ⚠️ ここにGASで「新しいデプロイ」をしたときに発行されたURLを貼り付けてください
    // -----------------------------------------------------------------
    const GAS_URL = "https://script.google.com/macros/s/{ユニーク文字列}/exec";
```

### 3. 当日の運用フロー（データの同期方法）
事前準備: 自宅や楽屋など、電波のある場所でタイマーのURLを開きます。

自動同期: 起動時にGASから最新のタイムテーブルが自動でスマホ（LocalStorage）にダウンロードされます。

本番: 一度読み込めば、ステージ上で機内モード（圏外）にしても、アプリを落とさない限り（またはキャッシュから）そのまま正常に動き続けます。

## 📱 スマホへのインストール方法・動作環境

### ⚠️ ローカル環境での実行に関する注意点
PCのローカル上で `index.html` ファイルを直接ダブルクリックして開いた場合（URLが `file:///...` になる状態）、ブラウザのセキュリティ制限（CORSポリシー）により、**GASからのタイムテーブル取得に必ず失敗します（これは正常な挙動です）。**

動作テストや本番運用を行う際は、 **GitHub Pagesなどのホスティングサービスを利用してWeb上に公開（HTTPS通信）する** か、VS Codeの `Live Server` 等を利用してローカルサーバー環境（`http://localhost:...`）を立ち上げてアクセスしてください。

### 📥 インストール手順
1. GitHub Pages等でホストし、発行された公開URL（`https://...`）をAndroidのChrome（またはiOSのSafari）で開きます。
2. 画面が正常に表示されたら、ブラウザのメニュー（右上または右下の「︙」ボタンなど）を開きます。
3. **「アプリをインストール」** または **「ホーム画面に追加」** をタップします。
4. ホーム画面に独立したアイコンが追加され、次からはURLバーのない全画面アプリとして起動できます。
